Servlet Demo

Mapping exceptions

  • /Error -> 500 Error
  • /Errore -> 404 Error

Cookies

  • /CreateCookie -> Creates a cookie with sample values
  • /EditCookie -> Edits the cookie created before
  • /DeleteCookie -> Deletes the cookie created before
  • /CookieInfo -> Displays information about the cookie

Sessions

  • /StartSession -> Sets some default attributes for the session
  • /EditSession -> Changes the session attributes
  • /DeleteSession -> Invalidates the session
  • /DeleteAttribute -> Just removes the 'edited' attribute

Filters example

  1. /GetLogin.html
    1. Enter Sophie:A -> Invalid login
    2. Enter Seb:Password -> Banned user
    3. Enter James:Password -> Logged in
  2. /DemoInput.jsp
    1. Enter James1 -> Error message asking for valid input
    2. Enter James -> Passed, displays your input

Listeners

The listeners I have implemented all output to the admin console of the server what's happening
  1. /AttributeTrigger -> Will create, change and delete an attribute
Listeners in place:
  1. Attribute Listener -> Will trigger whenever an attribute is created, changed or deleted
  2. Context Listener -> Triggers when the ServletContext lifecycle changes
  3. Request Listener -> Triggers when a servlet request is created or destroyed
  4. Session Listener -> Triggers when a session is created or invalidated

Redirecting

  1. Forward -> Forwards the request to another location (Will keep parameter values)
    • The extendedDemo system is using forwarding to keep your username when you login
  2. Redirect -> Redirected to a different source with a new request
    • The session and cookie examples are using redirect to get back to the information page
